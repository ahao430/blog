## 在编程工具中使用
### 安装CLI和桌面端
这里需要先安装nodejs环境: [https://nodejs.cn/](https://nodejs.cn/).

命令行安装claude code和codex。

```bash
# 安装claude code
npm install -g @anthropic-ai/claude-code

# 安装codex
npm i -g @openai/codex@latest

# opencode
npm i -g opencode-ai@latest
```

还可以安装codex桌面端

[https://openai.com/zh-Hans-CN/codex/get-started/](https://openai.com/zh-Hans-CN/codex/get-started/)

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1780831241230-0d680cfe-dc6e-406d-9c07-f4093d0fd75b.png)

claude code、codex、codex桌面端都需要官方账号购买订阅，授权登录。可以通过修改环境变量来使用第三方模型。

claude桌面端不使用环境变量，并且对第三方模型不友好，就不推荐了。喜欢用桌面端就用codex桌面端或者opencode桌面端。



zcode桌面端：[https://zcode.z.ai/cn/newdocs](https://zcode.z.ai/cn/newdocs)

智谱新出的桌面端，使用智谱glm模型时，用自家的工具，优化更好。

### 安装CC Switch
这里我们用CC Switch这个工具来方便配置多个第三方模型，一键切换环境变量使用不同模型。

下载最新版的CC Switch: [https://github.com/farion1231/cc-switch/releases](https://github.com/farion1231/cc-switch/releases) 。软件支持多平台，下载对应平台版本安装。

安装后界面如下，支持管理Claude Code、Codex、Gemini Cli、Opencode等工具， 用不到的可以在设置隐藏。

默认是选中claude code，这里刚安装好默认配置列表是空的。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772791070663-2bb9c20e-f04d-4e11-ba5d-22d60ada8f3e.png)

**我们先点击左上角设置按钮，找到“跳过Claude Code初次安装确认”，这个一定要打开。**否则登录claude的时候，就会弹出anthropic官方的登录，无法使用第三方模型。

这里如果设置了没有生效，退出claude，再打开试试。或者直接找到~/.claude.json, 在里面添加一行"hasCompletedOnboarding": true

上面的“应用到Claude Code插件”，如果你习惯用VSCode的claude code插件的话，可以打开，如果习惯用终端就无所谓了。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772791202302-55ca015e-8713-4865-8ec6-f3954a12b061.png)



### 在CC switch配置大模型
接下来回到主界面，点击右上角的加号。此时弹出一个添加供应商的弹窗。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772791327282-143482e1-53ce-4639-8298-e875d2dbe6e2.png)

如果我们买的某些官方服务，可以快速选择某个预设供应商模板，填入sk即可。

这里我们用自定义配置，往下拉。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772791375869-e7419257-3f05-42a2-826a-71d7d4288006.png)

供应商名称是给自己看的，随便写。官网链接可以不写。

重点是下面的API Key和请求地址。这里分别填写你的服务商令牌和请求地址。地址后面不要带/v1。

API格式这里默认是Anthropic的格式，如果你的模型不兼容Anthropic格式，可以选OpenAI格式。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772791498888-ef4bdc4d-dbec-45d8-9070-ed4a01bc0d1f.png)

这里主模型填上你的模型名称。点击添加保存。

保存后再次查看，发现这里几个模型全都是刚刚填写的主模型。

这里简单说一下，claude code在执行简单任务时，会调用claude haiku模型；一般任务会调用claude sonnet模型；复杂任务会调用claude opus模型。这里如果你的供应商提供了很多模型，你也可以给这几个地方分别配置上不同的模型，节约成本。比如买智谱编程套餐，官方就推荐opus填写glm-5,  sonnet填写glm-4.7，haiku填glm-4.5-air。

在列表点击配置的启用按钮。

打开任意一个终端，输入claude，回车。

可以看到以下界面，如果界面上的模型名称已经变成了你自定义的模型，说明成功了。

### ![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772791770966-7e9f0438-cf7b-457a-b2fd-b60e4cefebca.png)
### 在终端或IDE使用
#### 在项目中使用
claude会把当前启动的目录，当做项目目录。我们要在项目根路径的终端启动claude命令。

启动后，项目目录中会多一个.claude目录。其中是一些项目本地的claude设置项。

#### 在终端使用
在项目根目录，启动终端。输入claude命令。这不依赖于任何IDE。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772803226680-fe8fbf9d-8c3a-45c8-8b0e-35da5f9caa31.png)

#### vscode插件
如果不习惯终端界面，anthropic官方也提供了一个vscode插件。在插件市场搜索claude code。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772793581276-9d38526e-65b1-4405-b6d6-966db2c00f8d.png)

安装后，有两种使用方式。

1是找到vscode自带的聊天面板，这里是copilot。安装插件后，可以点击更多按钮，切换成claude code。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1773310570884-dc180a5d-1e89-49f4-a585-1bedb8eb667f.png)![](https://cdn.nlark.com/yuque/0/2026/png/373268/1773310592209-e1d640c3-fb3e-4e47-9545-56fc1cd92d0c.png)

2是打开任意一个文件，就可以看到文件右上方有一个claude的图标。点击图标，打开claude code窗口。

![](https://cdn.nlark.com/yuque/0/2026/png/373268/1772793635888-76fa71bc-b6dd-4903-97e0-dce9603a0c80.png)
