## 什么是 systemd
**systemd** 是 Linux 系统里的一个 **系统管理器和服务管理工具**。  
简单理解：

👉 **它负责启动、停止和管理系统里的各种服务。**

例如：

+ Web 服务器（nginx）
+ 数据库（mysql）
+ Docker

这些程序如果作为 **后台服务运行**，通常都是由 **systemd 管理**。

### systemd 可以做什么
#### 1️⃣ 启动服务
```plain
systemctl start nginx
```

#### 2️⃣ 停止服务
```plain
systemctl stop nginx
```

#### 3️⃣ 重启服务
```plain
systemctl restart nginx
```

#### 4️⃣ 查看状态
```plain
systemctl status nginx
```

---

### 什么是 service 文件
比如安装cliproxyapi，之后找到～/cliproxyapi目录下的cliproxy.service文件。

```plain
cliproxyapi.service
```

这是 **systemd 的服务配置文件**。

一般放在：

```plain
/etc/systemd/system/
```

里面会定义：

+ 程序在哪里
+ 如何启动
+ 是否开机启动
+ 用哪个用户运行

例如：

```plain
[Unit]
Description=Cliproxy API Service
After=network.target

[Service]
ExecStart=/usr/local/bin/cliproxyapi
Restart=always

[Install]
WantedBy=multi-user.target
```

---

### 为什么部署程序要用 systemd
好处很多：

✅ **开机自动启动**  
✅ **程序崩了自动重启**  
✅ **统一管理日志**  
✅ **后台运行**  
✅ **方便运维**

---

## 将服务安装到systemd
```plain
cd ~/cliproxyapi
sudo cp -f cliproxyapi.service /etc/systemd/system/cliproxyapi.service
sudo systemctl daemon-reload
```

检查它是否被systemd识别：

```plain
systemctl status cliproxyapi.service --no-pager
```

### 设置服务自动重启
#### <font style="color:rgb(34, 34, 34);">启动+开机自启</font>
```plain
sudo systemctl enable --now cliproxyapi.service
systemctl status cliproxyapi.service --no-pager
```

<font style="color:rgb(34, 34, 34);">看日志：</font>

```plain
sudo journalctl -u cliproxyapi.service -e --no-pager
```

## 查看服务
### 1️⃣ 查看当前正在运行的服务
```plain
systemctl list-units --type=service
```

只显示 **正在运行的 service**。

---

#### 2️⃣ 查看所有服务（包含未运行）
```plain
systemctl list-unit-files --type=service
```

会看到类似：

```plain
nginx.service           enabled
docker.service          enabled
cliproxyapi.service     enabled
ssh.service             enabled
```

状态说明：

+ **enabled** → 开机启动
+ **disabled** → 不开机启动
+ **static** → 依赖服务

---

#### 3️⃣ 查看所有正在运行的服务（最常用）
```plain
systemctl --type=service --state=running
```

---

#### 4️⃣ 搜索某个服务
比如查你的 `cliproxyapi`

```plain
systemctl list-unit-files | grep cliproxyapi
```

或

```plain
systemctl list-units | grep cliproxyapi
```

---

#### 5️⃣ 查看某个服务详细状态
```plain
systemctl status cliproxyapi
```

---

#### 6️⃣ 查看系统所有 unit（包括 socket、timer 等）
```plain
systemctl list-units
```
